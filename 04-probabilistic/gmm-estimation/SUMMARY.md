## The concept

Unlike hard clustering (every point belongs to exactly one group), GMM
models data as generated by a mixture of several Gaussian distributions,
each with its own mean, covariance (shape/spread), and weight (how common
that cluster is overall). GMM gives **soft assignments** -- a probability of
belonging to each cluster, not a single hard label, since a point between
two cluster centers genuinely has meaningful probability under both.

Training uses **EM (Expectation-Maximization)** -- the same algorithm behind
`04-probabilistic/missing-data-em/`: an E-step computes each point's
probability of belonging to each component given current parameters, an
M-step re-estimates each component's mean/covariance/weight given those soft
assignments, repeated until convergence.

## Experiment 1: 3D synthetic data, 4 known Gaussian components

The fitted GMM correctly recovered all 4 clusters, each with its own
ellipsoid showing fitted mean and covariance shape. One ellipsoid (centered
near the origin) is visibly broader/more stretched than the others -- this
matches the actual generating parameters in the code (that cluster has a
notably larger variance along one axis, 0.08 vs. 0.01 for the others), so
the GMM correctly recovered a real asymmetry in the data, not an error.

## Experiment 2: BIC/AIC model selection (how many components to use?)

Same "sweet spot" question as k in KNN or max_depth in Decision Trees --
except scored with an information-theoretic penalty (BIC/AIC) instead of a
validation set, and here verifiable against known ground truth: the
synthetic data generator used exactly 3 true Gaussian clusters.

```
Both BIC and AIC bottom out at n_components=3 -- the correct answer.
```

Neither metric was told the true number of clusters; both independently
found it purely by scoring fit-vs-complexity across n=1 to 9. Past n=3,
**BIC rises steadily and fairly monotonically** (its complexity penalty is
stronger, firmly rejecting unnecessary extra components), while **AIC
wiggles up and down** (dipping again around n=7) since its weaker penalty
tolerates slightly more complex models that marginally improve fit even past
the true structure. This is the standard, well-documented distinction
between the two criteria: BIC tends to prefer simpler models than AIC,
especially as more components are considered.
